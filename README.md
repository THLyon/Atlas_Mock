The system takes in large legal documents, intelligently chunks them into sentences, paragraphs, and sections using MiniLM and GPT-3.5, then embeds those at each level for retrieval. It stores this data in Pinecone (dense) and MongoDB (raw text and BM25 for sparse).
At query time, it parses the user's query to determine intent and granularity, dynamically re-chunks if needed, and retrieves the most relevant info through a hybrid search. It then compresses the result on the fly before sending the final summarized answer.
